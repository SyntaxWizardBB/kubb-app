# Feature note: Tournament scoring & rounds

Captured 2026-05-02 during the Lamport-clock ADR discussion. Lukas described how tournament-level scoring works above the individual match level. Belongs in M2 (Tournament-Setup) and shapes the data model for `tournament/`.

## Scoring modes (organizer chooses per tournament)

The organizer configures the scoring mode at tournament setup. Two modes for v1:

### Mode A: Wins counted

- Each match has a result like 2:1 (sets/games won by each team)
- Tournament standings = sum of match wins per team
- Simple to compute, easy for casual tournaments

### Mode B: Swiss system (Schweizer Modus)

- Match wins yield points
- Tiebreaker via Buchholz score (sum of opponents' scores)
- Used for tournaments where head-to-head matchups don't cover everyone

Both modes apply to the **group/preliminary phase**. The KO phase uses standard knockout rules independent of the group scoring mode.

## Group structure

- Morning groups can be variable in size (tournament-dependent)
- A typical day might have 7 matches per team in the group phase
- Group composition is set at tournament start; can be regenerated by organizer if registrations change before kickoff

## KO phase variants

Bracket structure varies by tournament:

- 1/8 final → 1/4 final → 1/2 final → final
- 1/4 final → 1/2 final → final
- Other configurations possible

The organizer picks the KO format at tournament setup. Must support: number of teams advancing from groups, seeding rules, and bracket-vs-double-elimination choice.

## Data model implication

Per Lukas: "es benötigt eigentlich pro match und pro tournier pro team" (regarding scoring aggregation).

Translated: the data model needs:

- **Per match**: result of that single match (e.g., 2:1)
- **Per tournament per team**: aggregated standings (wins, points, Buchholz, head-to-head, current rank)

Standings are derived from match results, not stored separately as truth — but should probably be cached/materialized for performance, since they are read on every player's match view.

## Open questions for when this feature is planned

- Buchholz computation: which variant? (Median Buchholz, Cut-1, full Buchholz?)
- Tiebreaker hierarchy when both Wins and Buchholz tie: head-to-head, then what?
- Can a tournament mid-flight change scoring mode? (probably no, but explicit)
- Bye handling in odd-numbered Swiss rounds?
- KO seeding rules: top group rank vs. snake seeding vs. organizer-manual?
